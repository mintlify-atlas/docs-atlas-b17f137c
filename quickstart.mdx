Welcome to Relio! This guide will walk you through creating your first workspace and adding your first contact in just a few minutes.

## Sign Up and Create Your Workspace

<Steps>
  <Step title="Create your account">
    Navigate to your Relio instance and sign up with your email address. After verifying your email, you'll be taken to the onboarding flow.
  </Step>

  <Step title="Set up your workspace">
    Create your first workspace by providing a name and slug. The workspace is your organization's central hub where you'll manage all your contacts, deals, and properties.

    ```typescript
    // Behind the scenes, Relio creates an organization
    const { data: org } = await authClient.organization.create({
      name: "Acme Real Estate",
      slug: "acme-real-estate"
    });
    ```

    <Tip>
    Choose a memorable slug - it will be part of your workspace URL: `https://your-domain.com/acme-real-estate`
    </Tip>
  </Step>

  <Step title="Invite your team (optional)">
    You can invite team members right away or skip this step and do it later from workspace settings.
  </Step>

  <Step title="Choose your plan">
    Select between the Free plan or start a Pro trial. You can change your plan anytime from the billing settings.

    - **Free**: Up to 1,000 contacts, basic features
    - **Pro**: Unlimited contacts, AI features, advanced integrations
  </Step>
</Steps>

## Add Your First Contact

Now let's add your first contact to your workspace.

<Steps>
  <Step title="Navigate to Contacts">
    From your workspace dashboard, click on **Contacts** in the sidebar or navigate to `/{your-workspace-slug}/contacts`.
  </Step>

  <Step title="Click 'Add Contact'">
    Click the **Add Contact** button in the top right corner of the contacts table.
  </Step>

  <Step title="Fill in contact details">
    Enter the contact's information. Only first and last name are required, but you can add much more:

    ```typescript
    // Example contact creation via API
    const contact = await trpc.contacts.create.mutate({
      firstName: "John",
      lastName: "Smith",
      email: [
        { 
          address: "john.smith@example.com", 
          isPrimary: true 
        }
      ],
      phone: [
        { 
          number: "+1-555-0123", 
          label: "mobile", 
          isPrimary: true 
        }
      ],
      role: "buyer",
      stage: "lead",
      connectionStrength: "strong"
    });
    ```

    **Available contact fields:**

    <CodeGroup>
    ```typescript Basic Info
    {
      firstName: string;        // Required
      lastName: string;         // Required
      title?: string;           // Job title
      avatarUrl?: string;
      description?: string;     // Notes about the contact
    }
    ```

    ```typescript Contact Details
    {
      email?: Array<{
        address: string;
        label?: string;
        isPrimary?: boolean;
        isBad?: boolean;
      }>;
      phone?: Array<{
        number: string;
        label?: string;
        isPrimary?: boolean;
        isBad?: boolean;
      }>;
      address?: Array<{
        street: string;
        street2?: string;
        city?: string;
        state?: string;
        zip?: string;
        label?: string;
        isPrimary?: boolean;
      }>;
    }
    ```

    ```typescript Relationship Fields
    {
      role: "buyer" | "seller" | "investor" | "agent" | "lender" | "other";
      stage: "lead" | "prospect" | "client" | "past_client";
      source?: "referral" | "website" | "social" | "event" | "import" | "other";
      connectionStrength: "weak" | "moderate" | "strong";
      ownerId?: string;  // Assign to team member
    }
    ```

    ```typescript Social & Location
    {
      linkedIn?: string;
      instagram?: string;
      facebook?: string;
      twitter?: string;
      angelList?: string;
      primaryLocation?: string;
      latitude?: number;
      longitude?: number;
    }
    ```

    ```typescript Buyer Needs
    {
      buyerNeeds?: {
        propertyType?: "residential" | "commercial" | "land" | "industrial" | "multi_family";
        minUnits?: number;
        maxUnits?: number;
        minPrice?: number;    // In dollars
        maxPrice?: number;
        minEquity?: number;   // Percent
        maxEquity?: number;   // Percent
        yearBuiltFrom?: number;
        yearBuiltTo?: number;
        exchange?: boolean;   // 1031 exchange preference
        cap?: number;         // Cap rate percent
      };
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Save and view your contact">
    Click **Save** to create the contact. You'll be taken to the contact detail page where you can:

    - Add notes and activities
    - Schedule calls and meetings
    - Link related contacts, companies, and properties
    - Upload attachments
    - Track interaction history
  </Step>
</Steps>

## Working with the API

Relio is built on a type-safe tRPC API. Here's how to interact with contacts programmatically:

```typescript
import { trpc } from "@/utils/trpc";

// List all contacts in your workspace
const contacts = await trpc.contacts.list.query({
  limit: 50,
  offset: 0,
  filters: {
    role: "buyer",
    stage: "lead"
  }
});

// Get a specific contact
const contact = await trpc.contacts.getById.query({
  id: "contact_id_here"
});

// Update a contact
const updated = await trpc.contacts.update.mutate({
  id: "contact_id_here",
  stage: "prospect",
  connectionStrength: "strong"
});

// Delete a contact
await trpc.contacts.delete.mutate({
  id: "contact_id_here"
});
```

<Warning>
All API operations are scoped to your current workspace. Make sure you have the correct workspace selected in your session.
</Warning>

## Real-Time Collaboration

Relio includes built-in real-time collaboration features powered by Supabase Realtime:

- **Live presence**: See who else is online in your workspace
- **Field locking**: When someone edits a field, it's locked for others
- **Typing indicators**: See when team members are composing notes
- **Activity updates**: New notes and activities appear instantly

<Tip>
To enable real-time features, make sure you've configured the optional Supabase environment variables. See the [installation guide](/installation#optional-real-time-collaboration) for details.
</Tip>
