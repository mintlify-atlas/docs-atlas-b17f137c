## Overview

Territories enable geographic organization of your sales and service areas. Define custom boundaries on the map, assign team members, and track coverage and performance metrics. Built on PostGIS spatial database technology, territories provide powerful location-based insights.

<Info>
  Territory features require properties and contacts to have valid latitude/longitude coordinates.
</Info>

## Creating Territories

### Drawing Boundaries

<Steps>
  <Step title="Access Territory Mode">
    Navigate to the map view and click "Territory" in the draw tools
  </Step>
  
  <Step title="Draw on Map">
    Click points on the map to create vertices of your territory boundary
  </Step>
  
  <Step title="Complete Polygon">
    Double-click to close the polygon or click the first point
  </Step>
  
  <Step title="Configure Territory">
    Set name, color, description, and assignment mode
  </Step>
</Steps>

### API Creation

```typescript
const territory = await trpc.territories.create.mutate({
  name: "Downtown District",
  color: "#3b82f6",
  description: "City center coverage area including financial district",
  boundary: {
    type: "Polygon",
    coordinates: [[
      [-122.4194, 37.7749],
      [-122.4194, 37.7849],
      [-122.4094, 37.7849],
      [-122.4094, 37.7749],
      [-122.4194, 37.7749]
    ]]
  },
  isExclusive: true
});
```

**Boundary Format:**
Territories use GeoJSON geometry format (Polygon or MultiPolygon) with coordinates in `[longitude, latitude]` order.

### Territory Properties

<ResponseField name="name" type="string" required>
  Territory display name (e.g., "North Region", "Downtown District")
</ResponseField>

<ResponseField name="color" type="string" required>
  Hex color code for map visualization (e.g., "#3b82f6")
</ResponseField>

<ResponseField name="description" type="string">
  Optional details about the territory coverage and purpose
</ResponseField>

<ResponseField name="boundary" type="GeoJSON" required>
  Polygon or MultiPolygon geometry defining the territory area
</ResponseField>

<ResponseField name="isExclusive" type="boolean" default="false">
  If true, only one team member can be assigned. If false, multiple members can share the territory.
</ResponseField>

## Assigning Team Members

### Exclusive Territories

For exclusive territories, only one member can be assigned at a time:

```typescript
// Assign to a single user (replaces previous assignment)
await trpc.territories.assign.mutate({
  territoryId: "territory_id",
  userIds: ["user_id"]
});
```

<Warning>
  Attempting to assign multiple users to an exclusive territory will result in an error.
</Warning>

### Shared Territories

For non-exclusive territories, assign multiple team members:

```typescript
// Assign multiple users to shared territory
await trpc.territories.assign.mutate({
  territoryId: "territory_id",
  userIds: ["user_1", "user_2", "user_3"]
});
```

**Assignment Logic:**
- Exclusive territories replace all existing assignments
- Shared territories add new assignments without removing existing ones
- Duplicate assignments are ignored (no duplicates created)

### Unassigning Members

```typescript
await trpc.territories.unassign.mutate({
  territoryId: "territory_id",
  userId: "user_id"
});
```

## Coverage Analysis

### Territory Coverage Metrics

Analyze what's inside each territory:

```typescript
const coverage = await trpc.territories.coverage.query({
  territoryIds: ["territory_1", "territory_2"] // optional filter
});

coverage.forEach(result => {
  console.log(result.territoryName);
  console.log(`  Properties: ${result.propertyCount}`);
  console.log(`  Contacts: ${result.contactCount}`);
  console.log(`  Deals: ${result.dealCount}`);
});
```

**Spatial Analysis:**
Uses PostGIS `ST_Within` function to determine which records fall within territory boundaries based on their latitude/longitude coordinates.

### Finding Coverage Gaps

Identify properties not covered by any territory:

```typescript
const gaps = await trpc.territories.gaps.query({
  limit: 100 // optional, defaults to 100
});

console.log(`${gaps.length} properties without territory coverage`);

gaps.forEach(property => {
  console.log(`${property.address}, ${property.city}`);
  console.log(`  Location: ${property.latitude}, ${property.longitude}`);
});
```

<Note>
  Gap analysis unions all territory boundaries and finds properties outside the combined area. Properties without coordinates are not included.
</Note>

## Performance Metrics

### Member Performance by Territory

Track team member activity and results within their assigned territories:

```typescript
const performance = await trpc.territories.memberPerformance.query();

performance.forEach(result => {
  console.log(`\n${result.territoryName}`);
  console.log(`Properties in territory: ${result.propertyCount}`);
  
  result.members.forEach(member => {
    console.log(`\n  ${member.userName}:`);
    console.log(`    Activities: ${member.activityCount}`);
    console.log(`    Deals: ${member.dealCount}`);
    console.log(`    Deal Value: $${member.dealValue.toLocaleString()}`);
  });
});
```

**Metrics Calculated:**
- Activity count (activities on properties in territory)
- Deal count (deals on properties in territory)
- Total deal value (sum of deal values)
- Per-member breakdown

<Tip>
  Use performance metrics to identify top performers, areas needing support, and territory rebalancing opportunities.
</Tip>

## Updating Territories

### Modify Territory Details

```typescript
await trpc.territories.update.mutate({
  id: "territory_id",
  name: "Updated Name",
  description: "Updated description",
  color: "#10b981",
  boundary: {
    // Updated boundary geometry
    type: "Polygon",
    coordinates: [/* new coordinates */]
  },
  isExclusive: false // Change to shared
});
```

**Boundary Updates:**
Changing the boundary automatically recalculates which properties and contacts fall within the territory.

### Exclusivity Changes

Converting between exclusive and shared:

```typescript
// Change to exclusive (will require removing extra assignees)
await trpc.territories.update.mutate({
  id: "territory_id",
  isExclusive: true
});

// Ensure only one assignee
const territory = await trpc.territories.getById.query("territory_id");
if (territory.assignees.length > 1) {
  // Remove all but one
  const keepUserId = territory.assignees[0].userId;
  for (let i = 1; i < territory.assignees.length; i++) {
    await trpc.territories.unassign.mutate({
      territoryId: "territory_id",
      userId: territory.assignees[i].userId
    });
  }
}
```

## Deleting Territories

<Warning>
  Deleting a territory removes all assignments and cannot be undone. Properties and contacts are not affected.
</Warning>

```typescript
// Admin only
await trpc.territories.delete.mutate("territory_id");
```

**Cascading Effects:**
- All territory assignments are deleted
- Territory boundary is removed from map
- Historical data (activities, deals) remains unchanged
- Properties/contacts are unaffected

## Spatial Queries

### Understanding PostGIS Integration

Territories use PostGIS spatial functions for powerful geometric operations:

**ST_Within:**
Determines if a point is inside a polygon
```sql
ST_Within(
  ST_SetSRID(ST_MakePoint(longitude, latitude), 4326),
  territory_boundary
)
```

**ST_MakeValid:**
Ensures boundary geometries are valid
```sql
ST_MakeValid(ST_SetSRID(ST_GeomFromGeoJSON(boundary), 4326))
```

**ST_Union:**
Combines multiple territories for gap analysis
```sql
ST_Union(boundary1, boundary2, ...)
```

### Coordinate System

All spatial data uses WGS 84 (SRID 4326):
- Longitude: -180 to 180 (East-West)
- Latitude: -90 to 90 (North-South)
- Format: `[longitude, latitude]`

await trpc.territories.create.mutate({

- Non-contiguous geographic areas
- Multiple cities or neighborhoods

## Troubleshooting

<AccordionGroup>
  <Accordion title="Properties Not Appearing in Territory">
    - Verify properties have valid latitude/longitude coordinates
    - Check that coordinates are in correct order: `[longitude, latitude]`
    - Ensure boundary polygon is valid (no self-intersections)
    - Confirm territory boundary is in WGS 84 (SRID 4326)
  </Accordion>
  
  <Accordion title="Coverage Gaps Not Found">
    - Ensure at least one territory exists
    - Verify properties have non-null coordinates
    - Check that PostGIS extension is enabled in database
    - Review territory boundaries for complete coverage
  </Accordion>
  
  <Accordion title="Assignment Conflicts">
    - Verify territory exclusivity setting
    - Check that all userIds are valid workspace members
    - Ensure admin permissions for assignment operations
    - Review existing assignments before modifying
  </Accordion>
</AccordionGroup>
