## Overview

Deals represent opportunities in your sales pipeline - property sales, acquisitions, leases, partnerships, or any transaction you're working on. Relio's deal management helps you track progress, forecast revenue, and close more deals.

## Creating Deals

<Steps>
  <Step title="Navigate to Deals">
    Click "Deals" in the main navigation
  </Step>
  
  <Step title="Create New Deal">
    Click "+ New Deal" in the top right
  </Step>
  
  <Step title="Enter Deal Details">
    Name the deal and set the initial stage
  </Step>
  
  <Step title="Link Records">
    Optionally link to a contact, property, or company
  </Step>
  
  <Step title="Add Financial Info">
    Set deal value, probability, and expected close date
  </Step>

## Deal Fields

### Basic Information

- **Name**: Descriptive deal name (e.g., "123 Main St Acquisition")
- **Emoji**: Optional emoji for visual identification
- **Description**: Deal notes and context
- **Owner**: Team member responsible for the deal

### Pipeline & Stage

Deals move through customizable pipeline stages:

```typescript
{
  pipelineId: "default_pipeline",
  stageId: "qualification",
  stageEnteredAt: "2024-03-13T10:30:00Z",

```

## Deal Pipelines

### Default Pipeline

Each workspace has a default sales pipeline with stages:

1. **Qualification** (active): Initial opportunity assessment
2. **Proposal** (active): Preparing and presenting offers
3. **Negotiation** (active): Discussing terms and conditions
4. **Closing** (active): Final paperwork and closing process
5. **Won** (won): Deal successfully closed
6. **Lost** (lost): Deal fell through or lost to competitor

### Stage Types

- **Active**: Deal is in progress
- **Won**: Deal closed successfully
- **Lost**: Deal did not close

Stage type determines:
- Whether `closedAt` is set automatically
- Inclusion in active deal metrics
- Win/loss rate calculations

### Custom Pipelines

Create custom pipelines for different deal types:

- Sales pipeline for property sales
- Acquisition pipeline for buying properties
- Leasing pipeline for tenant deals
- Partnership pipeline for business development

Each pipeline can have custom stages, colors, and progression logic.

<Tip>
  Contact your workspace admin to create custom pipelines for your team's workflow.
</Tip>

## Deal Permissions

Control who can view and edit deals:

### Visibility Settings

- **Private**: Only the deal owner and admins can see it
- **Workspace**: All workspace members can see it
- **Selected Users**: Specific users you choose

### Access Levels

For selected users:

- **View**: Can see deal details but not edit
- **Full Access**: Can view and edit deal details
- **No Access**: Cannot see the deal

### Managing Permissions

## Working with Deals

### Moving Through Stages

Update deal stage as progress happens:

```typescript
await trpc.deals.update.mutate({
  id: "deal_id",
  stageId: "negotiation_stage_id"
});
```

When you move a deal:
- `stageEnteredAt` is automatically updated
- If moved to won/lost stage, `closedAt` is set
- Activity log records the stage change

### Updating Probability

Adjust win probability as deal progresses:

```typescript
await trpc.deals.update.mutate({
  id: "deal_id",
  probability: 75  // 75% likely to close
});
```

Higher probability deals contribute more to weighted pipeline value.

### Expected Close Date

Set or update target close date:

```typescript
await trpc.deals.update.mutate({
  id: "deal_id",
  expectedCloseDate: new Date("2024-06-30")
});
```

Use expected close dates for:
- Revenue forecasting by period
- Prioritizing urgent deals
- Capacity planning
- Commission projections

## API Reference

Key deal operations:

```typescript
// List deals
const { items, total } = await trpc.deals.list.query({
  limit: 50,
  offset: 0,
  stageId: "qualification",
  ownerId: "user_id",
  sortField: "expectedCloseDate",
  sortDirection: "asc"
});

// Get pipeline view
const pipeline = await trpc.deals.pipeline.query();

// Get analytics
const analytics = await trpc.deals.analytics.query();

// Get single deal
const deal = await trpc.deals.getById.query("deal_id");

// Create deal
const newDeal = await trpc.deals.create.mutate({
  name: "123 Main St Acquisition",
  stageId: "qualification",
  value: 1250000,
  probability: 50,
  expectedCloseDate: new Date("2024-06-30"),
  contactId: "contact_id",
  propertyId: "property_id"
});

// Update deal
const updated = await trpc.deals.update.mutate({
  id: "deal_id",
  stageId: "negotiation",
  probability: 75
});

// Manage permissions
await trpc.deals.updateShares.mutate({
  dealId: "deal_id",
  visibility: "selected_users",
  shares: [
    { userId: "user_1", accessLevel: "full_access" },
    { userId: "user_2", accessLevel: "view" }
  ]
});
```
