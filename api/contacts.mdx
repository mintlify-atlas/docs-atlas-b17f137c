## Overview

The contacts router manages individual people in your CRM, including their contact information, relationships, and activity history.

## Data Structure

<ResponseField name="id" type="string" required>
  Unique contact identifier (CUID2)
</ResponseField>

<ResponseField name="workspaceId" type="string" required>
  Workspace this contact belongs to
</ResponseField>

<ResponseField name="firstName" type="string">

</ResponseField>

<ResponseField name="title" type="string">

## List Contacts

```typescript contacts.list
const result = await client.contacts.list.query({
  limit: 50,
  offset: 0,
  search: 'john',
  role: 'client',
  stage: 'qualified',
  sort: { field: 'createdAt', direction: 'desc' },
});
```

<ParamField path="limit" type="number" default="50">
  Number of records to return (max 100)
</ParamField>

<ParamField path="offset" type="number" default="0">
  Number of records to skip for pagination
</ParamField>

<ParamField path="cursor" type="string">
  Legacy cursor-based pagination ID
</ParamField>

<ParamField path="search" type="string">
  Search by first name or last name
</ParamField>

<ParamField path="firstName" type="string">
  Filter by first name (partial match)
</ParamField>

<ParamField path="lastName" type="string">
  Filter by last name (partial match)
</ParamField>

<ParamField path="title" type="string">
  Filter by job title (partial match)
</ParamField>

<ParamField path="role" type="enum">
  Filter by role: `lead`, `client`, `partner`, `vendor`, `other`
</ParamField>

<ParamField path="stage" type="enum">
  Filter by stage: `new`, `contacted`, `qualified`, `proposal`, `negotiation`, `closed_won`, `closed_lost`
</ParamField>

<ParamField path="source" type="enum">
  Filter by source
</ParamField>

<ParamField path="connectionStrength" type="enum">
  Filter by connection strength
</ParamField>

<ParamField path="ownerId" type="string">
  Filter by owner user ID
</ParamField>

<ParamField path="ownerPropertyIds" type="string[]">
  Filter contacts linked to properties as owners
</ParamField>

<ParamField path="listId" type="string">
  Filter by list membership
</ParamField>

<ParamField path="tagIds" type="string[]">
  Filter by tags (contacts must have ALL specified tags)
</ParamField>

<ParamField path="sort" type="object">
  Sort configuration
  <Expandable title="Sort object">
    <ParamField path="field" type="enum" default="createdAt">
      Sort field: `firstName`, `lastName`, `role`, `stage`, `source`, `connectionStrength`, `createdAt`
    </ParamField>
    <ParamField path="direction" type="enum" default="desc">
      Sort direction: `asc`, `desc`
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="items" type="array" required>
  Array of contact objects
</ResponseField>

<ResponseField name="total" type="number" required>
  Total count matching filters
</ResponseField>

<ResponseField name="nextCursor" type="string">
  Cursor for next page (legacy pagination)
</ResponseField>

### Auto-Enrichment

When `autoEnrichContacts` is enabled in workspace settings and a PDL API key is configured, new contacts with email addresses are automatically enriched in the background.

```typescript contacts.update
  id: 'contact_id',

```

<ParamField path="id" type="string" required>
  Contact ID to update

## Delete Contact

```typescript contacts.delete
await client.contacts.delete.mutate('contact_id');
```

<Warning>
  Requires admin or owner role. Deletes the contact and all related activities.
</Warning>

## Export Contacts

```typescript contacts.exportAll
const contacts = await client.contacts.exportAll.query({
  role: 'client',
  tagIds: ['tag1', 'tag2'],
});
```

Exports up to 5,000 contacts matching filters. Accepts same filters as `list` (except pagination).

<Warning>
  This is a heavy operation. Use pagination for regular queries.
</Warning>

## Enrich Contact

```typescript contacts.enrich
const result = await client.contacts.enrich.mutate({
  id: 'contact_id',
});
```

Enriches contact data from People Data Labs using the contact's email address. Falls back to Exa web search if PDL returns no results.

<ResponseField name="contact" type="object" required>
  Updated contact object
</ResponseField>

<ResponseField name="enrichedFields" type="string[]" required>
  List of fields that were populated
</ResponseField>

<ResponseField name="source" type="enum">
  Data source: `pdl` or `exa`
</ResponseField>

<ResponseField name="exaResults" type="array">
  Web search results from Exa (when PDL fails)
</ResponseField>

<Note>
  Requires PDL_API_KEY configuration. This is a heavy operation.
</Note>

## Connection Score

### Recalculate Strength

```typescript contacts.recalculateStrength
const contact = await client.contacts.recalculateStrength.mutate('contact_id');
```

Recalculates connection score and strength based on recent activity.

### Recalculate All

```typescript contacts.recalculateAllStrengths
const result = await client.contacts.recalculateAllStrengths.mutate();
```

Recalculates connection scores for all contacts in the workspace.

<Warning>
  Admin only. Use sparingly as this processes all contacts.
</Warning>

## Stale Contacts

```typescript contacts.getStaleContacts
const stale = await client.contacts.getStaleContacts.query({
  ownerId: 'user_id',
  limit: 25,
});
```

Returns contacts grouped by inactivity:
- **30 days**: No activity in 30-60 days
- **60 days**: No activity in 60-90 days  
- **90 days**: No activity in 90+ days or never

<ParamField path="ownerId" type="string">
  Filter by owner
</ParamField>

<ParamField path="limit" type="number" default="25">
  Max contacts per bucket (1-100)
</ParamField>

<ResponseField name="buckets" type="object">
  <Expandable title="Bucket structure">
    <ResponseField name="30" type="array">
      Contacts inactive 30-60 days
    </ResponseField>
    <ResponseField name="60" type="array">
      Contacts inactive 60-90 days
    </ResponseField>
    <ResponseField name="90" type="array">
      Contacts inactive 90+ days
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="counts" type="object">
  Count per bucket and total
</ResponseField>

## Aggregations

```typescript contacts.aggregations
const stats = await client.contacts.aggregations.query({
  role: 'client',
  aggregations: [
    { type: 'count' },
    { type: 'group', column: 'stage' },
    { type: 'group', column: 'tags' },
  ],
});
```

Perform analytics on contacts matching filters.

<ParamField path="aggregations" type="array" required>
  Array of aggregation operations
  <Expandable title="Aggregation types">
    <ParamField path="type" type="enum" required>
      `count`, `avg`, `sum`, `min`, `max`, `group`
    </ParamField>
    <ParamField path="column" type="string">
      Column to aggregate (required for all except `count`)
    </ParamField>
  </Expandable>
</ParamField>

Supported columns:
- `firstName`, `lastName`, `role`, `stage`, `source`, `connectionStrength`
- `twitterFollowerCount`, `createdAt`
- `tags` (junction table aggregation)

Returns aggregation results keyed by aggregation index.
