## Overview

The lists router manages collections of contacts, properties, companies, or deals with support for pipeline stages and permissions.

## Data Structure

<ResponseField name="id" type="string" required>
  Unique list identifier
</ResponseField>

<ResponseField name="workspaceId" type="string" required>
  Workspace this list belongs to
</ResponseField>

<ResponseField name="name" type="string" required>
  List name
</ResponseField>

<ResponseField name="emoji" type="string">
  Optional emoji icon
</ResponseField>

<ResponseField name="color" type="string">
  Hex color code for UI
</ResponseField>

<ResponseField name="description" type="string">
  List description
</ResponseField>

<ResponseField name="objectType" type="enum" required>
  Record type: `contact`, `property`, `company`, `deal`
</ResponseField>

<ResponseField name="type" type="enum">
  List type: `static`, `smart`
</ResponseField>

<ResponseField name="filterConfig" type="object">
  Filter configuration for smart lists
</ResponseField>

<ResponseField name="isPipeline" type="boolean">
  Whether this is a pipeline with stages
</ResponseField>

<ResponseField name="stages" type="array">
  Pipeline stages configuration
</ResponseField>

<ResponseField name="visibility" type="enum">
  Visibility: `workspace`, `private`, `shared`
</ResponseField>

<ResponseField name="workspaceAccessLevel" type="enum">
  Workspace access level: `view`, `full_access`
</ResponseField>

<ResponseField name="ownerId" type="string" required>
  List owner user ID
</ResponseField>

<ResponseField name="createdAt" type="datetime">
  Creation timestamp
</ResponseField>

## List All Lists

```typescript lists.list
const lists = await client.lists.list.query();
```

Returns all lists the current user has access to, with item counts.

<ResponseField name="lists" type="array">
  Array of list objects with:
  - `itemCount` - Number of items in the list
  - `ownerName`, `ownerImage` - Owner information
</ResponseField>

### Visibility Filtering

**Admins/Owners** see all workspace lists.

**Members** see:
- Workspace-visible lists (`visibility: 'workspace'`)
- Lists they own
- Lists explicitly shared with them

## List Items

### Get Items

```typescript lists.getItems
const result = await client.lists.getItems.query({
  listId: 'list_id',
  stageId: 'stage_id', // Optional: for pipeline lists
});
```

<ParamField path="listId" type="string" required>
  List ID
</ParamField>

<ParamField path="stageId" type="string">
  Filter by pipeline stage (for pipeline lists)
</ParamField>

<ResponseField name="list" type="object">
  List configuration
</ResponseField>

<ResponseField name="items" type="array">
  Array of list items with embedded record data:
  - `contact` - Full contact object (if contact list)
  - `property` - Full property object (if property list)
  - `company` - Full company object (if company list)
  - `deal` - Full deal object (if deal list)
</ResponseField>

### Add Item

```typescript lists.addItem
const item = await client.lists.addItem.mutate({
  listId: 'list_id',
  contactId: 'contact_id',
  stageId: 'stage_id', // Optional: for pipeline lists
  order: 0,
});
```

<ParamField path="listId" type="string" required>
  List ID
</ParamField>

<ParamField path="contactId" type="string">
  Contact ID (for contact lists)
</ParamField>

<ParamField path="propertyId" type="string">
  Property ID (for property lists)
</ParamField>

<ParamField path="companyId" type="string">
  Company ID (for company lists)
</ParamField>

<ParamField path="dealId" type="string">
  Deal ID (for deal lists)
</ParamField>

<ParamField path="stageId" type="string">
  Pipeline stage ID (for pipeline lists)
</ParamField>

<ParamField path="order" type="number" default="0">
  Display order within stage/list
</ParamField>

<Note>
  You must provide the ID matching the list's `objectType`. For example, a contact list requires `contactId`.
</Note>

</Note>

## Access Levels

<Accordion title="view">
  View-only access:
  - See list and its items
  - Cannot add, remove, or reorder items
  - Cannot modify list settings
</Accordion>

<Accordion title="full_access">
  Full access:
  - View list and items
  - Add and remove items
  - Reorder items
  - Update list settings
  - Cannot delete list (owner/admin only)
  - Cannot change permissions (owner/admin only)
</Accordion>

## Best Practices

<AccordionGroup>
  <Accordion title="Use meaningful names and emojis">
    Help users quickly identify lists:
    
    ```typescript
    {
      name: 'High-Value Opportunities',
      emoji: '💎',
      color: '#8B5CF6',
    }
    ```
  </Accordion>

  <Accordion title="Set appropriate visibility">
    - `workspace` - For shared sales pipeline, lead routing
    - `private` - For personal task lists, notes
    - `shared` - For team-specific lists with granular access
  </Accordion>

  <Accordion title="Use pipelines for workflows">
    Enable `isPipeline` and configure stages for visual workflows:
    
    ```typescript
    {
      name: 'Sales Pipeline',
      isPipeline: true,
      stages: [
        { name: 'Lead', color: '#6366F1' },
        { name: 'Qualified', color: '#3B82F6' },
        { name: 'Proposal', color: '#F59E0B' },
        { name: 'Closed', color: '#22C55E' },
      ],
    }
    ```
  </Accordion>

  <Accordion title="Bulk operations for performance">
    Use `bulkAddItems` instead of individual `addItem` calls:
    
    ```typescript
    // Good
    await client.lists.bulkAddItems.mutate({
      listId: 'list_id',
      entityIds: contacts.map(c => c.id),
    });
    
    // Avoid
    for (const contact of contacts) {
      await client.lists.addItem.mutate({
        listId: 'list_id',
        contactId: contact.id,
      });
    }
    ```
  </Accordion>
</AccordionGroup>
