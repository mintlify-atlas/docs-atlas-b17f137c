## Overview

The deals router manages sales opportunities through customizable pipelines with dynamic stages.

## Create Deal

```typescript deals.create
const deal = await client.deals.create.mutate({
  name: 'Office Lease - Acme Corp',
  value: 50000000, // $500k
  probability: 70,
  pipelineId: 'pipeline_id',
  stageId: 'stage_id',
  contactId: 'contact_id',
  propertyId: 'property_id',
  expectedCloseDate: new Date('2024-06-30'),
```

### Stage Automation

When creating a deal in a won/lost stage, `closedAt` is automatically set to the current timestamp.

## Update Deal

```typescript deals.update
const updated = await client.deals.update.mutate({
  id: 'deal_id',
  stageId: 'new_stage_id',
  probability: 85,
});
```

<ParamField path="id" type="string" required>
  Deal ID
</ParamField>

### Stage Change Automation

When `stageId` changes:
- `stageEnteredAt` is updated to current timestamp
- If moving to won/lost stage: `closedAt` is set
- If moving back to active stage: `closedAt` is cleared

### Update Shares

```typescript deals.updateShares
await client.deals.updateShares.mutate({
  dealId: 'deal_id',
  visibility: 'shared',
  workspaceAccessLevel: 'view',
  shares: [
    { userId: 'user1', accessLevel: 'full_access' },
    { userId: 'user2', accessLevel: 'view' },
  ],
});
```

<ParamField path="dealId" type="string" required>
  Deal ID
</ParamField>

<ParamField path="visibility" type="enum">
  Visibility: `workspace`, `private`, `shared`
</ParamField>

<ParamField path="workspaceAccessLevel" type="enum">
  Default workspace access: `view`, `full_access`
</ParamField>

<ParamField path="shares" type="array">
  Individual user shares (replaces all existing shares)
  <Expandable title="Share object">
    <ParamField path="userId" type="string" required>
      User ID
    </ParamField>
    <ParamField path="accessLevel" type="enum" required>
      Access level: `view`, `full_access`, `no_access`
    </ParamField>
  </Expandable>
</ParamField>

<Note>
  Only the deal owner or workspace admins can manage permissions.
</Note>

## Aggregations

```typescript deals.aggregations
const stats = await client.deals.aggregations.query({
  stageId: 'stage_id',
  aggregations: [
    { type: 'count' },
    { type: 'sum', column: 'value' },
    { type: 'avg', column: 'probability' },
    { type: 'group', column: 'stage' },
  ],
});
```

Supported columns:
- `name`, `stage`, `value`, `probability`
- `createdAt`, `expectedCloseDate`, `closedAt`
- `tags` (junction table)
