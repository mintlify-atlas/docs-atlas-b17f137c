## Overview

The properties router manages real estate listings, parcel data, and property information.

## Create Property

```typescript properties.create
const property = await client.properties.create.mutate({
  address: '123 Main St',
  city: 'San Francisco',
  state: 'CA',
  zip: '94102',
  propertyType: 'residential',
  status: 'for_sale',
  listPrice: 150000000, // $1.5M
  bedrooms: 3,
  bathrooms: 2,
  sqft: 2000,
});
```

When latitude/longitude are not provided, the property is automatically geocoded using Mapbox.

<ParamField path="address" type="string" required>
  Street address
</ParamField>

<ParamField path="city" type="string">
  City (geocoded if not provided)
</ParamField>

<ParamField path="state" type="string">
  State (geocoded if not provided)
</ParamField>

<ParamField path="zip" type="string">
  ZIP code (geocoded if not provided)
</ParamField>

<ParamField path="latitude" type="number">
  Latitude (geocoded if not provided)
</ParamField>

<ParamField path="longitude" type="number">
  Longitude (geocoded if not provided)
</ParamField>

All other property fields are optional.

## Enrich Property

```typescript properties.enrich
const result = await client.properties.enrich.mutate({
  id: 'property_id',
});
```

Enriches property data from multiple sources:
1. **Geocoding** (Mapbox) - If coordinates missing
2. **Walk Score API** - Walkability, transit, and bike scores
3. **Regrid** - Parcel data and property details
4. **Smarty** - Structural, mortgage, and market data

<ResponseField name="property" type="object">
  Updated property object
</ResponseField>

<ResponseField name="enriched" type="object">
  <Expandable title="Enrichment details">
    <ResponseField name="geocoded" type="boolean">
      Whether geocoding was performed
    </ResponseField>
    <ResponseField name="walkScore" type="number">
      Walk Score (0-100)
    </ResponseField>
    <ResponseField name="transitScore" type="number">
      Transit Score (0-100)
    </ResponseField>
    <ResponseField name="bikeScore" type="number">
      Bike Score (0-100)
    </ResponseField>
    <ResponseField name="walkScoreAvailable" type="boolean">
      Whether Walk Score data was fetched
    </ResponseField>
    <ResponseField name="regrid" type="object">
      Regrid enrichment details
    </ResponseField>
    <ResponseField name="smarty" type="object">
      Smarty enrichment details
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  This is a heavy operation. Requires API keys for enrichment sources.
</Note>

## Find by Parcel

```typescript properties.findByParcel
const match = await client.properties.findByParcel.query({
  llUuid: 'regrid_uuid',
  apn: '123-456-789',
  countyFips: '06075',
  latitude: 37.7749,
  longitude: -122.4194,
});
```

Finds a property using multiple matching strategies:
1. Regrid UUID (most reliable)
2. APN + County FIPS
3. Coordinate proximity (~50m)

<ResponseField name="id" type="string">
  Matched property ID
</ResponseField>

<ResponseField name="address" type="string">
  Property address
</ResponseField>

Returns `null` if no match found.

## Aggregations

```typescript properties.aggregations
const stats = await client.properties.aggregations.query({
  propertyType: 'residential',
  aggregations: [
    { type: 'count' },
    { type: 'avg', column: 'listPrice' },
    { type: 'group', column: 'city' },
    { type: 'sum', column: 'sqft' },
  ],
});
```

Supported columns:
- `listPrice`, `soldPrice`, `estimatedValue`, `bedrooms`, `bathrooms`, `units`
- `sqft`, `lotSqft`, `yearBuilt`, `equity`, `capRate`
- `assessedLandValue`, `assessedTotalValue`, `annualTax`
- `walkScore`, `transitScore`, `bikeScore`
- `address`, `city`, `state`, `propertyType`, `status`, `createdAt`
- `tags` (junction table)
