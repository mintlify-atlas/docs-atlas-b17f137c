## Overview

The tasks router manages workspace task management with assignees, linked records, and customizable columns.

## Data Structure

  Unique task identifier
</ResponseField>

<ResponseField name="workspaceId" type="string" required>

## List Tasks

```typescript tasks.list
const result = await client.tasks.list.query({
  limit: 50,
  status: 'todo',
  priority: 'high',
  assigneeId: 'user_id',
  contactId: 'contact_id',
  viewMode: 'mine', // or 'all'
  sortField: 'dueDate',
  sortDirection: 'asc',
});
```

<ParamField path="cursor" type="string">

<ParamField path="search" type="string">
  Search by task title
</ParamField>

<ParamField path="status" type="enum">
  Filter by status: `todo`, `in_progress`, `done`, `cancelled`
</ParamField>

<ParamField path="priority" type="enum">
  Filter by priority: `low`, `medium`, `high`, `urgent`
</ParamField>

<ParamField path="assigneeId" type="string">
  Filter by assigned user
</ParamField>

<ParamField path="contactId" type="string">
  Filter by linked contact
</ParamField>

<ParamField path="propertyId" type="string">
  Filter by linked property
</ParamField>

<ParamField path="companyId" type="string">
  Filter by linked company
</ParamField>

<ParamField path="dealId" type="string">
  Filter by linked deal
</ParamField>

<ParamField path="dueBefore" type="datetime">
  Filter by due date before timestamp

## List All Tasks

```typescript tasks.listAll
const tasks = await client.tasks.listAll.query();
```

Returns up to 500 tasks (non-paginated). Respects `viewMode` access control.

## Get Task by ID

```typescript tasks.getById
const task = await client.tasks.getById.query('task_id');
```

Members can only access tasks they created or are assigned to.

## Create Task

```typescript tasks.create
const task = await client.tasks.create.mutate({
  title: 'Follow up with prospect',
  description: '<p>Discuss pricing options</p>',
  status: 'todo',
  priority: 'high',
  dueDate: new Date('2024-03-25'),
  assigneeIds: ['user1', 'user2'],
  linkedRecords: [
    { recordId: 'contact_id', recordType: 'contact' },
    { recordId: 'deal_id', recordType: 'deal' },
  ],
});
```

<ParamField path="title" type="string" required>
  Task title
</ParamField>

<ParamField path="description" type="string">
  Task description (rich HTML)
</ParamField>

<ParamField path="status" type="enum" default="todo">
  Initial status
</ParamField>

<ParamField path="priority" type="enum">
  Task priority
</ParamField>

<ParamField path="dueDate" type="datetime">
  Due date
</ParamField>

<ParamField path="assigneeIds" type="string[]">
  User IDs to assign
</ParamField>

<ParamField path="linkedRecords" type="array">
  Records to link (new format, supports multiple)
  <Expandable title="Linked record object">
    <ParamField path="recordId" type="string" required>
      Record ID
    </ParamField>
    <ParamField path="recordType" type="enum" required>
      Type: `contact`, `property`, `company`, `deal`
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="contactId" type="string">
  Legacy single contact link (merged with linkedRecords)
</ParamField>

<ParamField path="propertyId" type="string">
  Legacy single property link
</ParamField>

<ParamField path="companyId" type="string">
  Legacy single company link
</ParamField>

<ParamField path="dealId" type="string">
  Legacy single deal link
</ParamField>

<Note>
  The first record of each type in `linkedRecords` is stored in the legacy single-link columns for backwards compatibility.
</Note>

## Update Task

```typescript tasks.update
const updated = await client.tasks.update.mutate({
  id: 'task_id',
  status: 'in_progress',
  priority: 'urgent',
  assigneeIds: ['user1', 'user2', 'user3'],
});
```

<ParamField path="id" type="string" required>
  Task ID
</ParamField>

<ParamField path="assigneeIds" type="string[]">
  Replace all assignees (pass empty array to clear)
</ParamField>

<ParamField path="linkedRecords" type="array">
  Replace all linked records
</ParamField>

All other fields are optional.

### Status Change Tracking

When `status` changes, `statusEnteredAt` is automatically updated to the current timestamp.

<Note>
  Members can only update tasks they created or are assigned to.
</Note>

## Delete Task

```typescript tasks.delete
await client.tasks.delete.mutate('task_id');
```

## Board View

```typescript tasks.board
const columns = await client.tasks.board.query();
```

Returns tasks grouped by status (Kanban board format):

<ResponseField name="columns" type="array">
  Array of status columns:
  <Expandable title="Column structure">
    <ResponseField name="status" type="enum">
      Status: `todo`, `in_progress`, `done`, `cancelled`
    </ResponseField>
    <ResponseField name="tasks" type="array">
      Tasks in this status
    </ResponseField>
    <ResponseField name="count" type="number">
      Number of tasks
    </ResponseField>
  </Expandable>
</ResponseField>

Tasks are sorted by position, then creation date (newest first).

## Column Preferences

Customize task board column appearance and behavior.

Returns column preferences keyed by status:

## Access Control

### Members

Regular members can:
- View tasks they created or are assigned to
- Create new tasks
- Update tasks they created or are assigned to
- Delete tasks they created or are assigned to

### Admins & Owners

Admins and owners can:
- View all workspace tasks
- Manage any task
- Access `viewMode: 'all'` in list queries

## Best Practices

<AccordionGroup>
  <Accordion title="Link tasks to records">
    Use `linkedRecords` to associate tasks with multiple CRM records:
    
    ```typescript
    linkedRecords: [
      { recordId: 'contact_id', recordType: 'contact' },
      { recordId: 'property_id', recordType: 'property' },
      { recordId: 'deal_id', recordType: 'deal' },
    ]
    ```
  </Accordion>

  <Accordion title="Assign collaboratively">
    Assign tasks to multiple team members for visibility:
    
    ```typescript
    assigneeIds: ['user1', 'user2']
    ```
  </Accordion>

  <Accordion title="Use priorities effectively">
    Set priorities to help team members focus:
    - `urgent` - Critical, needs immediate attention
    - `high` - Important, should be done soon
    - `medium` - Normal priority
    - `low` - Nice to have, can wait
  </Accordion>

  <Accordion title="Set due dates">
    Always set `dueDate` for time-sensitive tasks to enable deadline tracking.
  </Accordion>

  <Accordion title="Customize column behavior">
    Use column preferences to:
    - Track time in each status
    - Set target completion times
    - Celebrate wins with confetti
    - Hide unused statuses
  </Accordion>
</AccordionGroup>
