## Overview

The activities router tracks all interactions with contacts, properties, companies, and deals, including notes, calls, emails, meetings, and system events.

## Create Activity

```typescript activities.create
const activity = await client.activities.create.mutate({
  type: 'note',
  title: 'Follow-up call',
  body: '<p>Discussed next steps with <span class="mention" data-type="user" data-id="user123">@John Doe</span></p>',
  contactId: 'contact_id',
  dueAt: new Date('2024-03-20'),
});
```

### Mentions & Notifications

When creating activities with @mentions in the body:
- Mentioned users receive notifications
- Mentions are extracted from HTML `<span class="mention" data-type="user" data-id="...">` tags
- Only workspace members can be mentioned

### Contact Activity Tracking

When an activity is linked to a contact:
- Contact's `lastActivity` timestamp is updated
- Contact's connection score is recalculated

## Update Activity

  id: 'activity_id',
  body: '<p>Updated notes</p>',
  resolved: true,
});
```

<ParamField path="id" type="string" required>
  Activity ID

<ParamField path="body" type="string">
  Update content (only creator can edit)
</ParamField>

<ParamField path="resolved" type="boolean">
  Mark as resolved/unresolved (any user can resolve)
</ParamField>

<Note>
  Only the activity creator can edit the body. Anyone can resolve/unresolve.
</Note>

## Complete Activity

  id: 'activity_id',

```

## Delete Activity

```typescript activities.delete
await client.activities.delete.mutate({ id: 'activity_id' });
```

<Note>

</Note>

## Replies

### Create Reply

```typescript activities.createReply
const reply = await client.activities.createReply.mutate({
  activityId: 'activity_id',
  body: '<p>Great point! Let me follow up.</p>',
});
```

<ParamField path="activityId" type="string" required>
  Activity ID to reply to
</ParamField>

<ParamField path="body" type="string" required>
  Reply content (rich HTML, supports mentions)
</ParamField>

### Reply Notifications

When creating a reply:
- Mentioned users receive mention notifications
- Activity owner receives a notification (if not mentioned)
- Link includes `?highlightReply={replyId}` for deep linking

## Reactions

### Toggle Reaction

```typescript activities.toggleReaction
const result = await client.activities.toggleReaction.mutate({
  activityId: 'activity_id',
  emoji: '👍',
});
```

<ParamField path="activityId" type="string" required>
  Activity ID
</ParamField>

<ParamField path="emoji" type="string" required>
  Emoji character to react with
</ParamField>

<ResponseField name="action" type="enum">
  Result: `added` or `removed`
</ResponseField>

If reaction exists, it's removed. Otherwise, it's added.

### Reaction Notifications

When adding a reaction, the activity owner receives a notification (unless they're reacting to their own activity).

## Activity Types

<Accordion title="note">
  General notes and observations. Most flexible type for recording information.
</Accordion>

<Accordion title="call">
  Phone calls or video meetings. Can include duration in metadata.
</Accordion>

<Accordion title="email">
  Email correspondence. Email headers and thread info stored in metadata.
</Accordion>

<Accordion title="meeting">
  In-person or virtual meetings. Can include attendees and agenda in metadata.
</Accordion>

<Accordion title="task">
  Action items with due dates. Use `dueAt` field for scheduling.
</Accordion>

<Accordion title="system">
  System-generated events (e.g., record created, field updated). Created automatically by record activity logging.
</Accordion>

## Best Practices

<AccordionGroup>
  <Accordion title="Link activities to records">
    Always link activities to relevant contacts, properties, companies, or deals to maintain relationship context.
    
    ```typescript
    await client.activities.create.mutate({
      type: 'call',
      title: 'Discovery call',
      contactId: 'contact_id',
      propertyId: 'property_id', // Multi-link supported
    });
    ```
  </Accordion>

  <Accordion title="Use mentions for collaboration">
    Mention team members to notify them and keep everyone informed:
    
    ```typescript
    body: '<p><span class="mention" data-type="user" data-id="user123">@Alice</span> can you follow up on this?</p>'
    ```
  </Accordion>

  <Accordion title="Set due dates for tasks">
    Use the `dueAt` field with `type: 'task'` for actionable items:
    
    ```typescript
    await client.activities.create.mutate({
      type: 'task',
      title: 'Send proposal',
      dueAt: new Date('2024-03-25'),
      contactId: 'contact_id',
    });
    ```
  </Accordion>

  <Accordion title="Use metadata for structured data">
    Store additional context in the metadata field:
    
    ```typescript
    metadata: {
      duration: 1800, // 30 minutes
      outcome: 'scheduled_demo',
      nextSteps: ['send_proposal', 'book_demo'],
    }
    ```
  </Accordion>
</AccordionGroup>
